路径的写法

站内资源和站外资源

站内资源：当前网站的资源

站外资源：非当前网站的资源

绝对路径和相对路径

站外资源：使用绝对路径

站内资源：大部分使用相对路径

绝对路径的书写格式

协议名://主机名:端口号/路径
schema://host:port/path

协议名：http、https、file
主机名：域名、IP地址
端口号：如果协议是http协议,默认端口号80;如果协议是https协议,默认端口号443

当跳转目标和当前页面的协议相同时，可以省略协议

相对路径的书写格式

以./开头，./表示当前资源所在的目录

可以书写../表示

返回上一级目录

相对路径中，开头./可以省略

语义化

什么是语义化？

  1. 每一个HTML元素都具有具体的含义

    a元素：超链接

    p元素：段落

    h1元素：一级标题

  2. 所有元素与展示效果无关

元素展示到页面中的效果，应该由CSS决定。

因为浏览器带有默认的CSS样式，所以每个元素有一些默认的样式

💡

选择什么元素，取决于内容的含义，而不是显示出的效果

为什么需要语义化？

  1. 为了搜索引擎优化（SEO）⇒ 为了让搜索引擎更好的理解网页

    每隔一段时间，搜索引擎会从整个互联网中，抓取页面源代码，去分析这个网页拥有什么内容（结构、元素、内容含义），然后存在一起。去搜索的话，会更容易搜索到这些东西。

    语义化越好 ⇒ 搜索的结果越靠前

  2. 为了让浏览器理解网页

    浏览器一些特殊的功能，比如看小说网站，进入阅读模式会忽略菜单、广告等。语音模式（大声朗读）。为了让浏览器更好的理解哪些需要显示，哪些需要朗读，哪些需要用特殊的声调读。为了更好的适应其他功能，浏览器对网页理解的越深，浏览器应用这些功能就越好。

元素包含关系

以前：块级元素可以包含行级元素，行级元素不可以包含块级元素，a元素除外

现在：元素的包含关系由元素的内容类别决定。

例如：h1元素可以包含p元素

总结：

容器元素可以包含任何元素
a元素中几乎可以包含任何元素
某些元素有固定的子元素（ul>li，ol>li，dl>dt+dd）
标题元素和段落元素不能嵌套，并且不能包含容器元素

块级元素与行级元素的区别？

块级元素在页面中独占一行，而行级元素在页面中不换行。

列表元素

有序列表

ol：ordered list

li：list item

// type: 前面排序取值  1(阿拉伯),a(小写),A(大写),i(小写罗马),I(大写罗马)					
// 除非前面逻辑上很重要，一律不用 type ,用 css 属性写
// reversed 反序
<ol type="1" reversed>   
        <li>开门</li>
        <li>进去</li>
        <li>关门</li>
</ol>
效果:
    3. 关门
    2. 进去
    1. 开门

无序列表

ul：unordered list

li：list item

<ul>   
        <li>开门</li>
        <li>进去</li>
        <li>关门</li>
</ul>
效果:
    ●  开门
    ●  进去
    ●  关门

多媒体元素

video 视频

audio音频

用法几乎一模一样

video

<video controls="controls" src="xxx.mp4"></video>
<!-- 或 -->
<video controls src="xxx.mp4"></video>

controls：控制控件的显示，取值只能为controls

某些属性，只有两种状态：1.不写 2.取值为属性名，这种属性叫做布尔属性

布尔属性，在HTML5中，可以不用书写属性值

autoplay：布尔属性，进入网页后自动播放

muted：布尔属性，静音播放

loop：布尔属性，循环播放

audio

属性名、用法与video一致

兼容性

  1. 旧版本的浏览器不支持这两个元素

  2. 不同的浏览器支持的音视频格式可能不一致 ⇒ 有可能是.mp4,也有可能是.webm

    为了兼容性，一般不在video 写 src , 而在内部嵌套

    如以下代码：一旦mp4可以执行，则不执行后面webm

<video controls>
        <source src="xxx.mp4"></source>
        <source src="xxx.webm"></source>
</video>

图片元素

img元素

image缩写，空元素

src属性：source，图片路径

alt属性：当图片资源失效时，将使用该属性的文字替代图片

map元素

map：地图

<img usemap="#img" src="xxx">

<map name="img">
    <!-- 圆形 -->
    <area shape="circle" coords="x,y,radius" href="" alt="" target="">
    <!-- 矩形 -->
    <area shape="rect" coords="left-top-x,left-top-y,right-bottom-x,right-bottom-y" href="" alt="" target="">
    <!-- 多边形 -->	
    <area shape="poly" coords="a.x,a.y,b.x,b.y,......." href="" alt="" target="">
</map>

shape:形状
coords:坐标
href:路径
alt:失效时显示
target:跳转方式

figure元素

指代、定义，通常用于把图片、图片标题、描述包裹起来

子元素：figcaption

— 为了语义化，让浏览器明白这些内容是与包裹的图片相关联

— 建议将标题用figcaption 元素包裹起来，特指这块内容的标题

<figure>
    <figcaption>
        <h1>这是和图片关联的标题</h1>
    </figcaption>
    <p>这是和图片关联的内容</p>
    <img usemap="#img" src="xxx">
    <map name="img">
        <!-- 圆形 -->
        <area shape="circle" coords="x,y,radius" href="" alt="" target="">
        <!-- 矩形 -->
        <area shape="rect" coords="left-top-x,left-top-y,right-bottom-x,right-bottom-y" href="" alt="" target="">
        <!-- 多边形 -->	
        <area shape="poly" coords="a.x,a.y,b.x,b.y,......." href="" alt="" target="">
    </map>
</figure>

容器元素

容器元素：该元素代表一个块区域，内部用于放置其他元素

div元素

没有语义（浏览器无法识别这一块的内容）

<div>我表示一块区域</div>

语义化容器元素

header：通常用于表示页头，也可以表示文章的头部

<header>我表示一块头部区域</header>

footer：通常用于表示页脚，也可以表示文章的尾部

<footer>我表示一块尾部区域</footer>

article：通常用于表示整篇文章

<article>我表示一块文章区域</article>

section：通常用于表示文章的章节

<article>
        <section>
            <p>段落1</p>
        </section>
        <section>
            <p>段落2</p>
        </section>
        <section>
            <p>段落3</p>
        </section>
</article>

aside：通常用于表示侧边栏

<aside>我表示侧边栏</aside>

h

h1~h6：表示一级标题到6级标题

 h$*6>{$级标题}

span[无语义]

没语义，仅用于设置样式

三原色包含：
<span style="color: red">红</span>
<span style="color: green">绿</span>
<span style="color: blue">蓝</span>

以前：某些元素在显示时会独占一行（块级元素），而某些元素不会换行（行级元素)
到了HTML5，已经弃用这种说法。

pre

预格式化文本元素

空白折叠：在源代码中的连续空白字符 （空格、换行、制表），会被折叠成一个空格

例外：在pre元素中的内容不会出现空白折叠

在pre元素内部出现的内容，会按照代码格式显示到页面上

该元素通常用于在网页中显示一些代码

pre元素功能的本质：它有一个默认的css样式（white-space:pre;）

<p>Lorem,        
                    ipsum dolor.</p> 
=>Lorem, ipsum dolor.

<pre>Lorem,        
                    ipsum dolor.</pre>
=>Lorem,        
                    ipsum dolor.

通常在页面上显示代码时会在pre套一个code ,code 表示代码区域

    <code>
        <pre>
            var i = 0;
            i++;
        </pre>
    </code>
