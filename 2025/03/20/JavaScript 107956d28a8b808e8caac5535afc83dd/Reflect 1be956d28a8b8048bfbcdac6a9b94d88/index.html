Reflect

基础

let obj = {};

obj.a = 1; // [[SET]]
obj.a; // [[GET]]

当我们在对一个对象做一些基本操作的时候，我们本质上是就是在运行它的基本方法

也就是说，我们书写的语法层面的代码，它最终运行的时候，实际上运行的是这个对象的内部方法，对一个对象的所有操作都逃不过这一点。

💡

Reflect
调用对象的基本操作（内部方法）

⇒ Reflect 作用是不用去管语法，直接调用这个方法就完事了，直接调用对象的基本方法

// example
Reflect.set(obj, "a", 2);
Reflect.get(obj, "a");

那为什么有这种方法?

如果你是通过某一些语法或 API 间接地调用这个内部方法的话，他和直接调用内部方法是有区别的。当你使用某些语法去做一些事的时候，他会经过一系列的规则和步骤，这些规则和步骤种，其中某一步是再调用它的内部方法。

案例

当我们去执行proxy.c的时候，我们只会得到一次read:c ，但是因为 c 方法中我们访问了 this 点 a 和 this 点 b，但所以我们应该仍然需要触发 get 请求器，也就是获得 reda 和 redb，这个时候就会产生问题，我读 re DC 的时候我收集不到 reda 和 redb 的依赖

这个时候我们改用 proxy 改变了函数 c 中 this 的指向，即 this.a 和 this.b 的指向,所以我们去触发 Proxy.c 的时候，就会去触发 Proxy.a 和 Proxy.b

总结

在 proxy 里面操作对象时，最好采用 Reflect，避免出现 this 的指向问题
