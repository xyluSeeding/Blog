CommoJS 和 ESM 的区别

  1. 标准来源

CMJ：社区标准

由于是社区定义的，所以说有些内容它不能给你去新增，比如新增语法上。在语法层面吗，它是不敢去给你动的，它只能在 API 层面去动，所以搞出了一个函数叫做 require。所以 CMJ 其实没有带来什么新的语法，也新增了一个对象叫 module.exports

ESM：官方标准

来源不一样，就导致在落地层面不一样。
CMJ：新增 API
ESM：新增语法

  1. 时态

CMJ：运行态（运行过程中确定依赖关系，就是把模块放在函数里面）

if (xxx) {
  require(xxx);
} else {
  require(xxxx);
}

ESM：支持两种时态 运行时（import()） + 编译时 （静态） ⇒ tree shaking

import xxx from xxx

if (import.meta.env.MODE === "development") {
  const devTools = await import("./devTools.js");
  devTools.log("Debug mode");
} else {
  const prodLogger = await import("./prodLogger.js");
  prodLogger.log("User action");
}
